ccLab Simulator

This project is to create a game, and the content is open-ended. After reading the brief, I started trying to make demos for mini-games to gather inspiration – I created demos for a bouncing ball breaking bricks and a jumping ball avoiding obstacles. Then I tried to make these demos more visually appealing by uploading images to replace the bouncing ball and the jumping obstacle ball. I found the process of replacing the elements quite interesting, constantly experimenting with different elements to see them jump. Suddenly, I thought, why not turn this process of replacement into a game? This is when the idea for the cclab simulator came up – I want to create a pixel-style simulation of my game development process. I drew my workspace, and on the workspace, there’s my cat and various objects. By dragging them into my computer, I can replace the jumping ball or the life icon in the game. At the same time, the jumping obstacle game is still playable. The computer screen will also be the p5.js interface. So, I started making the game. I used Photoshop to create various pixel elements and assembled my desk, then exported the draggable elements into separate layers. I set them up as draggable objects. On the computer screen, I created a canvas within a canvas to hold the main game. When I dragged a draggable object into that small canvas, I replaced the ball or life icon inside the game with the icon of the dragged object, and the draggable item would disappear and refresh at the original position. The cat I drew is my friend's cat named Orange, so when I dragged the little Orange cat in, the life icon would change to the icon of Orange (which might be a bit confusing if the cat turns into an orange). Then, I created the initial and ending screens for the game. The initial screen shows some instructions and details about the assignment. The ending screen appears after completing and submitting the assignment. The process of designing the screens went smoothly, but the only thing that caused me some struggle was ensuring the screens appeared in sequence without conflicts. I spent some time figuring out the display order of the pages. My solution was to divide them into separate layers and edit them individually, then write the logic for when they should appear at the beginning. This part was a bit complex, and I had to think it through, but I managed to organize everything well, which gave me a great sense of accomplishment.

* All pixel illustrations and elements, except for the lava lamp, were created and edited by myself. The lava lamp GIF was created by a user named 'purplepill' and posted on the Lospec website (https://lospec.com/gallery/purplepill/lava-lamp).
